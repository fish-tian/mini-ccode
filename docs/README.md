# 教学文档

每个模块完成后产出教学文档。

教学文档不是 API 文档，而是解释思想。

## 模板

```md
# <模块名> 教学文档

## 这个模块解决什么问题

用通俗方式解释。

## 编程 Agent 工具为什么需要它

解释设计动机。

## 最小实现

从最小可运行版本讲起。

## 本项目中的实现

解释当前实现方案。

## 教学版取舍

重点说明当前 mini-ccode 实现与 生产级系统 的差异：

- 表面行为有什么不同。
- 架构边界有什么不同。
- 内部机制少了哪些生产能力。
- 这些简化为什么适合当前教学阶段。
- 后续要接近 生产级系统，需要补哪些模块。

## 关键代码导读

列出几个核心文件、类型或函数。

## 常见误区

- 把 agent loop 写成一次性 chat completion
- 工具调用没有状态机
- tool result 没有结构化
- 权限系统被工具绕过
- 上下文无限追加历史

## 可扩展方向

说明未来可以如何增强。
```
